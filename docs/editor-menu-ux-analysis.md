## 2. 문제점 분석

### [Critical] RSVP가 "설정"에 매몰

**현상**: RSVP(참석 여부 수집)가 `SettingsTab` 안에 비밀번호/자동삭제와 함께 묻혀있음
(`components/editor/tabs/SettingsTab.tsx` 261-355행)

**영향**:
- RSVP는 청첩장의 핵심 기능. 경쟁사(보닐, 오하루, 모두의청첩장) 모두 독립 섹션으로 운영
- "설정"까지 가보지 않는 사용자는 RSVP 존재 자체를 모를 수 있음
- RSVP 활성화율 저하 -> 사용자의 "이 플랫폼은 기능이 부족하다" 인식 가능성
- RSVP 데이터 수집은 하객 관리의 핵심이므로, 별도 노출이 필수

### [Important] AI 사진 기능의 노출 부족

**현상**: AI 사진 생성(핵심 차별화 포인트)이 갤러리 탭(5번째) 안에 포함
- 일반 사진 업로드와 AI 생성이 같은 탭에 혼재
- 사이드바에서 "갤러리"라는 이름만으로는 AI 기능의 존재를 알 수 없음

**영향**:
- AI가 이 플랫폼의 유일한 차별화인데, 사이드바에서 전혀 드러나지 않음
- "사진 올리는 곳" 정도로 인식되어 AI 기능 발견율(discoverability) 낮음
- 무료->프리미엄 전환의 핵심 동력인데 숨겨져 있으면 전환율에 직접 타격

### [Important] Settings 탭 과부하

**현상**: 설정에 성격이 다른 5가지가 혼재
- 섹션 순서 변경 (레이아웃)
- 비밀번호 보호 (접근 제어)
- RSVP 설정 (핵심 기능)
- 자동 삭제 안내 (정보)
- 통계 (정보)

**영향**: 사용자가 "설정은 나중에" 심리로 건너뛰면 RSVP와 섹션 순서 같은 중요 기능을 놓침

### [Minor] D-Day 달력 스타일이 예식장 탭에 위치

**현상**: D-Day 달력 스타일(없음/미니달력/카운트다운/미니멀)이 `VenueTab` 하단에 있음
(`components/editor/tabs/VenueTab.tsx` 396-424행)

이건 예식 "정보"가 아니라 "표시 방식" 설정 -> 템플릿 커스터마이징이나 설정에 더 적합

### [Minor] 인사말 탭이 너무 가벼움

**현상**: 인사말 탭은 textarea 1개 + 예시 3개뿐. 독립 탭으로는 콘텐츠가 빈약
- 사용자 입장에서 "한 탭인데 이것뿐?" 느낌 가능

## 3. 잘 되어있는 점

| 항목 | 평가 |
|------|------|
| 템플릿 맨 처음 | 업계 표준. 시각적 흥미로 첫 인상 좋음. 경쟁사들도 동일 |
| 필수->선택 순서 | 핵심부터 채우고 나머지는 자유롭게. 인지부하 줄임 |
| 기본 정보 -> 예식장 순서 | "누가" -> "언제/어디서" 자연스러운 흐름 |
| 인사말 -> 갤러리 -> 계좌 순서 | 감성(인사말) -> 시각(사진) -> 실용(계좌) 흐름 자연스러움 |
| Toggleable 선택 섹션 | 선택권 부여로 부담 감소. 좋은 패턴 |
| 완성도 표시 (체크/경고) | 진행 상황 직관적 파악. 게이미피케이션 효과 |
| Step Navigation (이전/다음) | 선형 흐름 지원. 모바일 전환 시에도 유용 |

## 4. 경쟁사 비교 (한국 모바일 청첩장 시장)

| 기능/섹션 | 보닐 | 오하루 | 모두의청첩장 | Cuggu(현재) |
|-----------|------|--------|-------------|-------------|
| 템플릿 선택 | 1번 | 1번 | 1번 | 1번 OK |
| 기본 정보 | 2번 | 2번 | 2번 | 2번 OK |
| 예식 정보 | 3번 | 3번 | 3번 | 3번 OK |
| 인사말 | 4번 | 4번 | 4번 | 4번 OK |
| 갤러리 | 5번 | 5번 | 5번 | 5번 OK |
| 계좌/축의금 | 6번 | 6번 | 6번 | 6번 OK |
| **RSVP** | **독립 탭** | **독립 탭** | **독립 탭** | **설정에 매몰** |
| AI 사진 | 없음 | 없음 | 없음 | 있지만 숨김 |

**핵심 인사이트**:
- 1~6번 순서는 업계 표준과 완벽히 일치. 변경 불필요
- RSVP만 경쟁사 대비 구조적으로 뒤처짐
- AI 사진은 경쟁사에 없는 유일한 차별점인데 더 적극적으로 노출해야 함

## 5. 제안: 개선된 메뉴 구조

### 변경 전 (7탭)
```
템플릿 -> 기본 정보 -> 예식장 -> 인사말 -> 갤러리 -> 계좌 -> 설정
```

### 변경 후 (8탭)
```
템플릿 -> 기본 정보 -> 예식장 -> 인사말 -> 갤러리 -> 계좌 -> RSVP -> 설정
```

### 최종 탭 구조

```
+-------------------------------------+
| (그룹 라벨 없음)                      |
|   템플릿 - 청첩장 디자인 선택          |
+-------------------------------------+
| 필수                                 |
|   기본 정보 - 신랑/신부 정보와 가족     |
|   예식장 - 날짜/장소/오시는 길         |
+-------------------------------------+
| 선택                                 |
|   인사말 - 하객들께 전하는 메시지  [토글]|
|   갤러리 - 사진 업로드/AI 사진   [토글] |
|   계좌 - 축의금 계좌 안내       [토글]  |
|   참석 확인 - 하객 참석 여부    [토글]  |  <- NEW
+-------------------------------------+
|   설정 - 공개 설정 및 부가 옵션        |  <- 경량화됨
+-------------------------------------+
```

## 6. 구체적 변경사항

### 변경 1: RSVP 독립 탭 분리 (Priority: Must)

**AS-IS**: Settings 탭 안의 RSVP 섹션 (261-355행)
**TO-BE**: 새로운 `RsvpTab` 컴포넌트, optional 그룹, toggleable

```typescript
// lib/editor/tabs.ts 변경
{ id: 'rsvp', label: '참석 확인', description: '하객 참석 여부 수집',
  icon: ClipboardCheck, toggleable: true, group: 'optional' },
```

**변경 파일**:
- `lib/editor/tabs.ts` - 탭 정의 추가
- `components/editor/tabs/RsvpTab.tsx` - 새 탭 컴포넌트 (SettingsTab에서 RSVP 부분 추출)
- `components/editor/tabs/SettingsTab.tsx` - RSVP 섹션 제거
- `components/editor/EditorPanel.tsx` - switch문에 rsvp case 추가
- `schemas/invitation.ts` - DEFAULT_ENABLED_SECTIONS에 rsvp 추가

**UX 효과**: 사이드바에서 바로 보이므로 RSVP 활성화율 대폭 상승 예상

### 변경 2: 갤러리 탭에 AI 차별화 강조 (Priority: Important)

사이드바 표시만 개선 (코드 구조 변경 최소화):

**AS-IS**: label: "갤러리", description: "사진 업로드 및 관리"
**TO-BE**: label: "갤러리", description: "사진 업로드 / AI 사진 생성" + AI 뱃지 표시 (선택)

**변경 파일**:
- `lib/editor/tabs.ts` - description 변경
- `components/editor/SectionPanel.tsx` - AI 뱃지 추가 (선택)

**비즈니스 효과**: AI 기능 발견율 상승 -> 프리미엄 전환 유도

### 변경 3: D-Day 달력 스타일을 설정으로 이동 (Priority: Nice-to-have)

**AS-IS**: VenueTab 하단에 D-Day 달력 스타일 선택
**TO-BE**: SettingsTab의 "표시 옵션" 섹션으로 이동

예식장 탭은 "정보 입력"에 집중, 표시 방식은 설정에서 관리하는게 관심사 분리에 맞음.
단, 이건 우선순위 낮음 - 현재 위치도 크게 문제는 아님.

## 8. 검증 방법

1. 에디터 페이지(`/editor/[id]`)에서 사이드바 8개 탭 정상 표시 확인
2. RSVP 탭 클릭 -> RSVP 설정 폼 정상 동작 확인
3. RSVP 토글 on/off -> enabledSections 반영 확인
4. Settings 탭에서 RSVP 제거 확인
5. 갤러리 탭 description에 AI 문구 표시 확인
6. Step Navigation (이전/다음)에서 8탭 순서대로 이동 확인
7. 기존 초대장 로드 시 RSVP 설정 데이터 유지 확인
