# 2026-02-03: 3가지 랜딩 페이지 버전 구현

## 작업한 내용

### 1. 3가지 랜딩 페이지 버전 구현
- **옵션 A: 엘레강트 웨딩** (`/landing-a`)
  - 꽃잎 떨어지는 애니메이션 (Framer Motion)
  - 골드/아이보리/블러쉬 핑크 색상 팔레트
  - 템플릿 캐러셀 (5개 템플릿, 자동/수동 슬라이드)
  - SVG 꽃 장식 요소
  - 전통적 웨딩 감성, 세리프 폰트

- **옵션 B: 모던 럭셔리** (`/landing-b`)
  - 블랙/화이트/골드 색상
  - 분할 레이아웃 (좌: 텍스트, 우: 이미지)
  - 템플릿 그리드 (6개, 호버 인터랙션)
  - 대담한 타이포그래피
  - 미니멀하고 고급스러운 느낌

- **옵션 C: 스토리텔링** (`/landing-c`)
  - 문제 → 해결 → 프로세스 구조
  - Before/After 비교 슬라이더
  - 스크롤 페이드 애니메이션
  - 따뜻한 파스텔 (핑크/라벤더)
  - 단계별 가이드, 감성적 접근

### 2. 공통 애니메이션 컴포넌트 개발
- `FallingPetals.tsx`: 꽃잎 떨어지는 애니메이션 (10개 랜덤 생성)
- `ScrollFade.tsx`: 스크롤 기반 페이드인 효과 (Intersection Observer)
- `ParallaxSection.tsx`: 패럴랙스 스크롤 효과

### 3. 기술 스택 추가
- `framer-motion@12.30.0`: 애니메이션 라이브러리
- `tailwindcss-animate@1.0.7`: Tailwind 애니메이션 유틸리티

### 4. 라우트 구조 개선
- 대시보드를 `app/(dashboard)/` → `app/admin/`으로 이동
- 라우트 그룹 제거하고 명확한 `/admin` 경로 사용
- DashboardNav 링크 업데이트 (`/dashboard` → `/admin`)

## 왜 했는지 (맥락)

### 문제 인식
- 기존 랜딩 페이지가 너무 밋밋함 (단순 그라데이션, SaaS 느낌)
- 웨딩/청첩장 감성 부족
- 애니메이션 없음, 시각적 임팩트 약함

### 목표
- 청첩장 플랫폼에 어울리는 **웨딩 감성** 표현
- 3가지 완전히 다른 디자인 버전 제공하여 선택지 확보
- Framer Motion을 활용한 부드러운 애니메이션 추가

### 디자인 전략
- **옵션 A**: 전통적 웨딩 감성 (한국 시장 친화적)
- **옵션 B**: 현대적 럭셔리 (젊은 층 타겟)
- **옵션 C**: 스토리텔링 (문제 해결 중심, 전환율 최적화)

## 논의/아이디어/고민

### 1. 라우트 그룹 vs 명확한 경로
**질문:** `(marketing)`과 `(dashboard)`의 차이는?
- 라우트 그룹은 URL에 나타나지 않고, 레이아웃 공유 목적
- `(dashboard)`는 레이아웃은 분리되지만 `/dashboard` 경로 필요

**결정:** `/admin` 경로로 명확하게 분리
- 이유 1: URL 구조가 더 명확함 (`/admin`은 관리자 영역 명시)
- 이유 2: 확장 용이 (`/admin/users`, `/admin/analytics` 등)
- 이유 3: 라우트 그룹 불필요 (폴더 자체가 이미 구분)

### 2. 모든 버전 유지 vs 하나만 선택
**결정:** 일단 모든 버전 유지
- 3개 버전 모두 커밋하여 언제든 재활용 가능
- 나중에 사용자 피드백 받아서 최종 선택

### 3. 플레이스홀더 이미지 사용
- Unsplash API 사용 (무료 웨딩 사진)
- 나중에 실제 템플릿 목업으로 교체 예정

## 느낀 점/난이도/발견

### 난이도: 중 (⭐⭐⭐☆☆)
- Framer Motion API는 직관적이지만, 최적화 고려 필요
- 3가지 완전히 다른 디자인을 빠르게 구현하는 것이 핵심
- Next.js 16 라우트 그룹 이해 필요했음

### 발견한 것들
1. **Next.js 16 라우트 충돌**
   - `(marketing)/page.tsx`와 `(dashboard)/page.tsx`가 같은 `/` 경로 충돌
   - 해결: 대시보드를 `/admin`으로 명확하게 분리

2. **Framer Motion 의존성 이슈**
   - 처음에 `framer-motion`이 devDependencies로 설치됨
   - Runtime에 필요하므로 dependencies로 재설치 필요

3. **tailwindcss-animate 누락**
   - 기존 `tailwind.config.ts`에서 참조했지만 설치 안 됨
   - 추가 설치 필요

### 성능 고려사항
- 꽃잎 애니메이션: 10개로 제한 (성능 vs 시각적 효과 균형)
- 스크롤 애니메이션: `once: true` 옵션 (한 번만 실행, 리렌더링 방지)
- 이미지: lazy loading 적용 (기본 Next.js Image 최적화)
