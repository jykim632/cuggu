# 청첩장 만들기 플로우 — 다중 역할 시뮬레이션 리뷰

> 날짜: 2026-02-10
> 목적: 청첩장 생성→편집→발행→공유→하객열람 전체 플로우를 5개 역할 관점에서 분석
> 방법: 코드베이스 전수 탐색 기반, 실제 구현 상태 반영

---

## 현재 플로우 요약

```
대시보드 → "새 청첩장 만들기" → POST /api/invitations (DRAFT)
  → /editor/{id} 리다이렉트 → 3패널 에디터 (탭 네비 | 폼 | 미리보기)
  → 8개 탭 편집 (2초 디바운스 자동저장)
  → "발행하기" (필수값 4개 검증) → /inv/{id} URL 복사 → 공유
```

**에디터 레이아웃** (~1400px 최소)
```
┌─────────────────────────────────────────────────────────┐
│ TopBar: 로고 | 제목 | 저장상태 | AI크레딧 | 버튼들      │
├──────────┬──────────────────────┬───────────────────────┤
│ Section  │    Editor Panel     │    Preview Panel      │
│ Panel    │    (폼 영역)         │    (실시간 미리보기)    │
│ 220px    │    flex-grow        │    420px              │
│ 8개 탭    │  이전/다음 네비게이션  │  모바일/데스크톱 토글   │
│ 토글 스위치│  max-w-3xl center  │  아이폰/갤럭시 선택     │
│ 상태 뱃지 │                     │  줌 슬라이더           │
└──────────┴──────────────────────┴───────────────────────┘
```

**8개 탭**

| 순서 | 탭 | 필수 | 내용 |
|------|------|------|------|
| 1 | 템플릿 | - | 디자인 선택 + AI 테마 생성 |
| 2 | 기본정보 | O | 신랑/신부 이름, 부모, 관계, displayMode |
| 3 | 예식장 | O | 날짜, 시간, 장소 (카카오맵 검색) |
| 4 | 인사말 | 토글 | 하객 메시지 (예시 3개 제공) |
| 5 | 갤러리 | 토글 | 사진 업로드(10장) + AI 사진 생성 |
| 6 | 계좌 | 토글 | 축의금 계좌 (신랑/신부/부모) |
| 7 | RSVP | 토글 | 참석 여부 수집 항목 설정 |
| 8 | 설정 | - | 섹션 순서(dnd-kit), 비밀번호, 폰트, 캘린더 스타일 |

## 1. PM / 비즈니스 관점

### 전환 퍼널 분석

```
가입 → 첫 청첩장 생성 → 편집 완료 → 발행 → 공유 → 하객 열람 → RSVP
  ↓         ↓              ↓          ↓       ↓         ↓          ↓
 🔴        🟢            🟡        🟡      🔴       🟡         🟡
```

| 단계 | 현재 상태 | 판단 |
|------|----------|------|
| **가입** | Registration Gate ON (신규가입 차단 중) | 의도적 베타. OK |
| **첫 생성** | 1클릭 생성, 기본값 자동입력 | **좋음**. 활성화 마찰 최소 |
| **편집 완료** | 8탭, 가이드 없음, 데스크톱 전용 | 이탈 위험 높음 |
| **발행** | 필수값 4개만 검증 후 즉시 발행 | 마찰 낮음. OK |
| **공유** | 링크 복사만. 카카오톡 에디터에서 바로 안됨 | **치명적 병목** |
| **하객 열람** | 공개 페이지 존재, OG 태그 있음 | 기능적으로 OK |
| **RSVP** | API 구현됨, 게스트 폼 존재 | OK |

### 수익 모델 평가

| 항목 | 현재 | 리스크 |
|------|------|--------|
| 무료 크레딧 | 2회 (가입 시) | 적정. AI 맛보기로 충분 |
| 단가 | ₩1,000/회 | 마진 72~89%. 건강함 |
| 번들 | 10회 ₩8,000 (20% 할인) | OK |
| 프리미엄 | ₩9,900 (10크레딧 포함) | 한번 구매형. 반복 매출 없음 |
| 템플릿 | 6개 전부 무료 | 프리미엄 템플릿 미출시 = 업셀 기회 없음 |
| 결제 | Toss 미구현 | **결제 못 받는 상태** |

### PM 핵심 지적사항

1. **분석 도구 전무** — GA, Mixpanel, PostHog 없음. 퍼널 데이터 수집 불가. 어디서 이탈하는지 모름
2. **결제 미구현** — 크레딧 구매 UI는 있는데 실제 결제(Toss) 연동 안 됨. 매출 $0
3. **공유 퍼널 끊김** — 에디터에서 카카오톡 직접 공유 불가 (공개 페이지 ShareBar에서만 가능). 발행 후 링크 복사 → 카카오톡 열기 → 붙여넣기 필요
4. **만료 자동화 없음** — `expiresAt` 컬럼은 있는데 상태를 EXPIRED로 바꾸는 크론잡/트리거 없음
5. **반복 매출 구조 없음** — 1회성 구매만. 구독 모델이나 재구매 유도 장치 부재

## 2. UI/UX 디자이너 관점

### A. 첫 진입 경험 (Critical)

**현재**: 생성 즉시 → 빈 8탭 에디터로 던져짐

- 온보딩 모달 없음
- 가이드 투어 없음
- 진행률 표시 없음
- "여기서 시작하세요" 하이라이트 없음

**문제**: 사용자가 "뭘 먼저 해야 하지?"를 스스로 파악해야 함. 탭이 8개나 있고 필수/선택 구분이 사이드바 그룹핑으로만 표현됨.

**제안**:
- 첫 생성 시 3단계 퀵 위자드 (이름 → 날짜/장소 → 템플릿 선택) → 이후 에디터로
- 또는 에디터 진입 시 "필수 정보부터 채워볼까요?" 가이드 오버레이

### B. 잘된 점

- 실시간 미리보기 (편집 즉시 반영)
- 디바이스별 미리보기 토글 (아이폰/갤럭시)
- 탭별 완료 상태 뱃지 (녹색 체크/빨간 경고)
- 선택 섹션 토글 스위치
- 2초 디바운스 자동저장
- @dnd-kit 드래그 섹션 정렬

### C. 문제점

| 이슈 | 심각도 | 설명 |
|------|--------|------|
| **모바일 편집 불가** | Critical | 3패널 고정 레이아웃. 반응형 전무. sm:/md: 브레이크포인트 없음 |
| **저장 실패 무음** | High | 자동저장 실패 시 localStorage 백업만. 사용자에게 알림 없음 |
| **필드 검증 부재** | Medium | 입력 중 실시간 검증 없음. 발행 시점에만 토스트로 알려줌 |
| **스켈레톤 없음** | Medium | 로딩 시 스피너만. 레이아웃 점프 발생 가능 |
| **빈 상태 빈약** | Medium | 갤러리, 계좌 등 빈 폼만 보여줌. 예시/가이드 부족 |

### D. 탭별 UX 상세

| 탭 | 잘된 점 | 개선 필요 |
|-----|---------|----------|
| **템플릿** | AI 테마 생성기 눈에 띄는 그래디언트 카드 | 6개 무료 템플릿만. 프리미엄 영역 비어있음 |
| **기본정보** | displayMode별 조건부 필드 노출, 故 체크박스 | 관계 셀렉터 → 커스텀 입력 전환 UX 어색할 수 있음 |
| **예식장** | 카카오 API 주소 검색, 지도 미리보기 | 검색 드롭다운 모바일 대응 안됨 |
| **인사말** | 3개 예시 문구 원클릭 삽입 | **가장 잘 된 탭**. 다른 탭도 이 패턴 적용 필요 |
| **갤러리** | 드래그앤드롭, 용량/개수 제한 명시 | AI 사진 생성이 갤러리 아래 묻혀있음 |
| **계좌** | 부모 계좌 접기/펼치기 | 빈 폼 검증 없음. 은행 선택 없이 저장 가능 |
| **RSVP** | 수집 항목 체크박스 커스텀 | 게스트 폼 미리보기 없음 ("하객에게 이렇게 보입니다" 부재) |
| **설정** | @dnd-kit 드래그 섹션 정렬 | 비활성 섹션 "숨김" 표시가 작아서 놓칠 수 있음 |

### E. 네비게이션

- 탭 자유 이동 가능 (검증 없이 점프)
- 상단/하단에 이전/다음 버튼 + 단계 카운터 (1/8)
- 비활성 탭은 카운트에서 제외 (토글 OFF 시)
- **없는 것**: 프로그레스 바, 브레드크럼, 키보드 단축키 안내

## 3. 콘텐츠 전략가 관점

### 현재 카피/문구 현황

| 위치 | 현재 문구 | 평가 |
|------|----------|------|
| 기본 신랑 이름 | "신랑" | 기능적. "이름을 입력하세요"가 나을 수 있음 |
| 기본 신부 이름 | "신부" | 동일 |
| 기본 예식장 | "예식장" | 동일 |
| 발행 성공 | "청첩장이 발행되었습니다! 링크가 복사되었습니다." | 기능적. 감성 없음 |
| 저장 상태 | "저장 중" / "방금 전" | OK |
| 비밀번호 게이트 | 자물쇠 아이콘 + 입력창 | 최소한. "소중한 분만 볼 수 있어요" 같은 카피 없음 |
| 만료 페이지 | "만료된 청첩장입니다" / "더 이상 열람할 수 없습니다" | 차가움. 결혼은 끝나도 추억은 남기에 |
| 에러 메시지 | "필수 정보를 입력해주세요: 신랑 이름, 신부 이름..." | 리스트형. 친절하지 않음 |

### 핵심 지적

1. **감성 부재** — 청첩장은 인생에서 가장 설레는 순간의 산물. 현재 톤은 "폼 빌더" 수준
2. **인사말 예시 3개만** — 경쟁사는 20~50개 카테고리별 제공 (종교, 유머, 격식 등)
3. **플레이스홀더가 데이터를 가장** — "신랑"이 기본값이라 미리보기에 "신랑 ♥ 신부"가 뜸. 실제 이름인지 기본값인지 구분 불가
4. **에러가 무미건조** — "신랑 이름 미입력" 대신 "신랑 이름을 알려주세요" 같은 톤
5. **공유 시 메시지 없음** — 링크만 복사됨. "결혼합니다! 소중한 분들께 전해주세요" 같은 동반 메시지 템플릿 없음
6. **OG 태그 기본 설명** — introMessage 첫 100자 또는 기본 인사말. 더 매력적인 기본값 필요

## 4. 실제 사용자 (예비 신랑/신부) 관점

### 시나리오: "다음 달 결혼하는 30대 커플"

#### 가입~생성

> "카카오 로그인했더니 바로 대시보드에 떨어졌네. 뭐 하라는 건지... 아, '새 청첩장 만들기' 버튼 보인다. 클릭."
>
> "오 바로 에디터가 열렸다. 근데 화면이 세 덩어리로 나뉘어있고 왼쪽에 탭이 8개... 좀 많다."

**Pain**: 안내 없이 에디터에 떨어짐. 8탭이 위압적.

#### 기본정보 입력

> "신랑 이름, 신부 이름... OK. 근데 '부모님 표시 방식'이 뭐지? 'full_names'? 아... 한복 씌워서 부모님 이름도 넣는 거구나. 우리 아빠 돌아가셨는데... 아, '故' 체크박스가 있네. 다행이다."
>
> "관계가 '장남'으로 되어있는데 난 둘째인데... 아 드롭다운에서 '차남' 선택하면 되는구나."

**Good**: 故 표시, displayMode 전환. 민감한 가족 상황 배려.
**Pain**: displayMode 라벨이 영어(full_names). 한글 설명 필요.

#### 예식장 입력

> "예식장 검색이 바로 되네. 카카오 맵이랑 연동되어 있어서 주소도 자동. 좋다."
>
> "근데 시간 선택이... 시간/분 따로 고르는 건 좀 귀찮다. 대부분 12시, 1시, 2시 정각/30분이니까 프리셋 있으면 좋겠다."

**Good**: 카카오 맵 검색 통합.
**Pain**: 시간 선택 UX.

#### 갤러리/AI 사진

> "사진 업로드는 쉽다. 10장까지 되고. 근데 AI 사진 생성은 뭐지? 크레딧이 2개 있다는데..."
>
> "오 AI로 사진을 만들어준다고? 해볼까. 스타일 고르고... 기다리고... 와 신기하다!"
>
> "근데 크레딧 다 쓰면 돈 내야 하는데, 결제 버튼 누르니까... 아무것도 안 되네?"

**Good**: AI 사진 생성은 강력한 "와" 포인트.
**Critical Pain**: 크레딧 구매 결제가 안 됨. 체험 후 구매 전환이 끊김.

#### 계좌 입력

> "축의금 계좌... 좀 민망하긴 한데 요즘 다 하니까. 신랑/신부 각각, 부모님 계좌도 넣을 수 있네."
>
> "근데 은행을 안 골랐는데도 저장이 되네? 나중에 빠트리면 어떡하지?"

**Pain**: 빈 폼 검증 없음. 불완전 저장 가능.

#### 발행 시도

> "'발행하기' 눌렀더니 '필수 정보를 입력해주세요: 예식장 이름'이라고 뜬다. 아, 아까 예식장 탭에서 이름을 안 넣었구나. 근데 진작 알려주지..."

**Pain**: 발행 시점에야 검증. 편집 중 피드백 없음.

#### 공유

> "발행됐다! 링크가 복사됐다는데... 이걸 카카오톡에 붙여넣기 해야 하나? 카톡 공유 버튼은 없나?"
>
> "에디터에서는 안 보이고... 아, 링크 열어보니까 거기에 카카오 공유 버튼이 있네. 내가 먼저 내 청첩장을 열어보고 거기서 공유해야 하는 거구나. 좀 돌아가는데?"

**Critical Pain**: 에디터 → 카카오톡 공유 직통 경로 없음. 발행 → 링크 복사 → 공개 페이지 열기 → 공유 버튼 클릭 = 3단계.

#### 모바일

> "집에서 컴퓨터로 만들다가 출근길에 폰으로 수정하려고 했는데... 화면이 다 깨진다. 세 패널이 겹치고 글자가 잘리고. 이건 PC에서만 되는 건가?"

**Critical Pain**: 모바일 편집 불가. 결혼 준비 중인 커플이 항상 PC 앞에 있지 않음.

## 5. 그로스 해커 / 마케터 관점

### 바이럴 루프 분석

```
Creator → 청첩장 생성 → 공유 → Guest 열람 → [바이럴 전환] → Guest가 Creator 됨
                                                    ↑
                                              이 전환이 핵심
```

**현재 바이럴 장치**:
- 공개 페이지 하단 "Cuggu" 브랜드 링크 (항상 노출)
- OG 태그로 카카오톡 미리보기 카드 생성
- ShareBar에서 카카오톡/링크/네이티브 공유

**바이럴 루프 병목**:

| 단계 | 현재 | 문제 |
|------|------|------|
| 공유 도달 | 링크 복사 or 카카오톡 | 에디터에서 카톡 직접 공유 없음 |
| 첫 노출 | OG 카드 미리보기 | 기본 설명 밋밋. "결혼합니다" 수준 |
| 재공유 | ShareBar (공개 페이지) | 하객이 다른 하객에게 공유할 동기 없음 |
| 전환 유도 | 하단 "Cuggu" 링크 | 클릭 추적 없음. CTA가 약함 ("나도 만들어보기" 아님) |
| 분석 | viewCount만 | 유입 경로, 공유 횟수, 재방문 추적 불가 |

### 핵심 지적

1. **분석 인프라 제로** — GA/Mixpanel/PostHog 없음. A/B 테스트 불가. 퍼널 최적화 불가
2. **UTM 파라미터 없음** — 어디서 유입됐는지 추적 불가
3. **공유 카운터 없음** — 몇 번 공유됐는지 모름
4. **CTA 미약** — 하단 "Cuggu" 대신 "나도 이렇게 예쁜 청첩장 만들기 →" 가 필요
5. **리퍼럴 시스템 없음** — 친구 초대 시 크레딧 보상 같은 장치 없음
6. **카카오톡 SDK 키 이슈** — 지도 API 키를 공유 SDK에도 재사용 중 (분리 필요할 수 있음)

### 그로스 우선순위 제안

| 우선순위 | 액션 | 예상 임팩트 |
|---------|------|-----------|
| **P0** | 분석 도구 도입 (GA4 or PostHog) | 모든 최적화의 전제조건 |
| **P0** | 에디터에서 카카오톡 직접 공유 | 공유 퍼널 단축 (3단계→1단계) |
| **P1** | 하단 CTA 강화 ("나도 만들기") | 바이럴 전환율 직접 영향 |
| **P1** | 공유 메시지 템플릿 | "결혼합니다" + 자동 메시지 첨부 |
| **P2** | 리퍼럴 프로그램 | 공유 → 가입 시 양쪽 크레딧 |
| **P2** | UTM 추적 | 채널별 효과 측정 |

## 종합: 역할별 Top 3 이슈

| 역할 | #1 | #2 | #3 |
|------|-----|-----|-----|
| **PM** | 결제 미구현 (매출 $0) | 분석 도구 없음 | 공유 퍼널 끊김 |
| **UI/UX** | 모바일 편집 불가 | 온보딩 부재 | 저장 실패 무음 |
| **콘텐츠** | 감성 톤 부재 | 인사말 예시 부족 | 에러 메시지 무미건조 |
| **사용자** | 카톡 공유 동선 복잡 | 발행 시점 뒤늦은 검증 | 모바일 사용 불가 |
| **그로스** | 분석 인프라 제로 | 바이럴 CTA 미약 | 리퍼럴 없음 |

### 역할 간 공통 교차점 (가장 임팩트 큰 개선 4가지)

1. **모바일 에디터** — UX/사용자/그로스 전부 지적. 한국 사용자 80%+ 모바일.
2. **카카오톡 공유 통합** — PM/사용자/그로스 전부 지적. 한국 시장 필수.
3. **분석 도구** — PM/그로스 지적. 이것 없이는 어떤 최적화도 블라인드.
4. **결제 연동** — PM 지적. 매출이 없으면 사업이 아님.
