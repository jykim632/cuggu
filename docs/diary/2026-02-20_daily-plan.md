## 1단계: 청첩장 핵심 빠진 것 채우기

### 1-1. 방명록 구현 — `cuggu-s2k` P1

한국 청첩장 필수 기능. 경쟁사 전부 제공. 없으면 "미완성" 느낌.

**스코프:**
- `guestbook_entries` 테이블 신설 (name, message, invitationId, isHidden, createdAt)
- API: `POST/GET /api/invitations/[id]/guestbook`
- 인증 불필요, IP 레이트리밋 (분당 3회)
- 주인이 부적절한 메시지 숨김 가능
- 최신순 정렬, 페이지네이션 (10개씩)
- 공개 뷰 + 에디터에서 on/off 토글

**예상:** 가장 큰 작업. DB + API + 공개뷰 UI + 에디터 설정.

### 1-2. 엔딩 섹션 — `cuggu-erp` P2

청첩장 끝이 허전함. 경쟁사 대비 '심각' 격차.

**스코프:**
- 푸터 바로 위에 전체 너비 사진 + 마무리 문구 섹션
- `ending: { photo?: string, message?: string }` 데이터
- 예시 문구 제공
- 고정 위치 (reorderable 아님)

**예상:** 비교적 가벼움. 템플릿 렌더러 + 에디터 탭 추가.

### 1-3. 콘텐츠 톤 개선 — `cuggu-nvr` P2

현재 "폼 빌더" 수준의 톤 → 감성 청첩장 톤으로 전환.

**스코프:**
- 인사말 예시 3개 → 15~20개 (격식/캐주얼/종교/유머/계절)
- 플레이스홀더: "신랑" → "이름을 입력하세요"
- 에러 톤: "필수 정보를 입력해주세요" → "조금만 더 채워주세요"
- 발행 성공: 감성적 축하 메시지
- 만료 페이지 / 비밀번호 게이트 톤 개선

**참고:** `docs/diary/2026-02-10_invitation-flow-multi-role-review.md` 콘텐츠 전략가 섹션

**예상:** 코드보다 카피 작업. 파일 여러 개 걸치지만 각각은 작음.

## 2단계: 공유 경험 강화

### 2-1. 커스텀 OG 이미지 — `cuggu-bpa` P2

카카오톡 공유 시 첫인상. 청첩장의 80%는 카톡으로 공유됨.

**스코프:**
- `share: { ogImage?: string, ogTitle?: string }` 에디터 설정
- `generateMetadata()`에서 커스텀 값 우선 적용
- 현재는 첫 갤러리 사진으로 자동 생성

**예상:** 0.5일.

### 2-2. 카카오 OG 캐시 초기화 — `cuggu-fwh` P2

OG 이미지 바꿔도 카카오톡에서 캐시가 안 풀리는 문제.

**스코프:**
- 카카오 스크래퍼 API 호출 (`POST https://developers.kakao.com/tool/debugger/sharing`)
- 청첩장 발행/수정 시 자동 호출 또는 에디터에서 수동 버튼
